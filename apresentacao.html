Comandos Git Essenciais

Principais comandos Git para controle de versão e fluxo de desenvolvimento

  • Configuração Inicial:
  • git config --global user.name "Seu Nome"
  • git config --global user.email "email@exemplo.com"
  • Comandos Básicos:
  • git init - Inicializar repositório
  • git clone <url> - Clonar repositório
  • git add . - Adicionar arquivos ao stage
  • git commit -m "mensagem" - Fazer commit
  • git push origin main - Enviar para GitHub
  • git pull origin main - Baixar atualizações
  • Branches:
  • git branch nome-branch - Criar branch
  • git checkout nome-branch - Trocar branch
  • git merge nome-branch - Fazer merge
  • Status e Histórico:
  • git status - Ver status dos arquivos
  • git log --oneline - Ver histórico
Documentação Git ↗ Tutorial Interativo Git Everyday

Segurança no Desenvolvimento

Práticas essenciais para proteger dados sensíveis e código

Fundamental • Sempre verificar

🔐
Arquivo .env

NUNCA coloque dados sensíveis diretamente no código. Use sempre arquivos .env para armazenar chaves de API, senhas e tokens.

  • API Keys - Chaves de serviços externos
  • Tokens - Autenticação e autorização
  • Senhas - Banco de dados e serviços
  • URLs privadas - Endpoints internos

Exemplo .env:

OPENAI_API_KEY=sk-proj-xxx...
DATABASE_URL=postgresql://user:pass@localhost:5432/db
JWT_SECRET=your-secret-key-here
🚫
.gitignore

SEMPRE verifique se o arquivo .env está no .gitignore para não subir dados sensíveis para o repositório.

  • .env* - Todos os arquivos de ambiente
  • node_modules/ - Dependências
  • *.log - Arquivos de log
  • dist/ - Arquivos compilados

Exemplo .gitignore:

# Variáveis de ambiente
.env
.env.local
.env.*.local

# Dependências
node_modules/

# Logs
*.log
🤖
Verificação por IA

SEMPRE peça para a IA verificar a segurança do código antes de fazer commit. IAs são excelentes para detectar vulnerabilidades.

  • Scan de segurança - Vulnerabilidades conhecidas
  • Dados expostos - Chaves e senhas no código
  • Validação de entrada - SQL injection, XSS
  • Permissões - Controle de acesso

Prompt para IA:

"Analise este código em busca de vulnerabilidades de segurança, dados sensíveis expostos e boas práticas não seguidas. Verifique se há chaves de API, senhas ou tokens hardcoded."

📝
Código Comentado

Mantenha o código bem comentado e claro para evitar que a IA "alucine" e introduza bugs ou vulnerabilidades.

  • Funções documentadas - O que fazem e por quê
  • Variáveis claras - Nomes descritivos
  • Lógica explicada - Comentários em partes complexas
  • TODOs marcados - Melhorias futuras

Exemplo de bom comentário:

// Valida e sanitiza entrada do usuário para prevenir XSS
function sanitizeInput(userInput) {
  // Remove tags HTML perigosas
  return userInput.replace(/<script>/gi, '');
}
⚠️

Regra de Ouro da Segurança

"Nunca confie, sempre verifique" - Mesmo com IA, sempre revise o código gerado. Configure seu ambiente para detectar automaticamente dados sensíveis e use ferramentas como git-secrets ou truffleHog para escanear repositórios. Segurança não é opcional, é fundamental!
